# 8.5 Applications of Queues
## Chapter Overview:
- A queue is a linear data structure that follows FIFO (First In, First Out)
    - The element added first is removed first
- Insertions happen at the end and deletions happen at the beginning
- Works similar to a grocery store line: the first in line is served first

## Key Concepts:
- Implementation of Queues:
    - Can be stored in memory using:
        - Arrays
        - Linked Lists
- Types of Queues:
    - Linear Queues:
        - Usually implemented using arrays
        - One problem is a wasted space after deletion because as the front moves forward, the freed space can't be reused
    - Circular Queues:
        - The last index connects back to index 0 (last position is connected to the first position), which solves the memory wastage problem of linear queues
            - Is an efficient use of array space
    - Linked Queues:
        - Implemented using a linked list where the size is not fixed
            - Memory usage grows as needed and enqueue/dequeue operations have a time complexity of O(1)
    - Deque (Double-Ended Queue): 
        - Elements can be inserted or deleted on either end, but there are no operations allowed in the middle
        - Implemented using:
            - Circular Array
            - Circular Doubly Linked List
        - Types:
            - Input-restricted deque: insertion at one end, deletion at both
            - Output-restricted deque: deletion at one end, insertion at both 
    - Priority Queue:
        - Every element is assigned a priority and are processed based on that priority
            - If priorities are equal then the elements follow FIFO
        - Every node contains:
            - Data
            - Priority Value
            - Link to next node
- Application of Queues:
    - Can be used in:
        - CPU Scheduling
        - Printer and Disk Management
        - I/O System Buffers
        - Music Playlists
        - Josephus Problem
- Josephus Problem:
    - Follows the idea that people stand in a circle, and every k-th person is eliminated, continuing until only one person remains. 
        - This can be solved using a circular linked list or queue

## Review Questions
### 1. What is a priority queue? Give its applications.
    A priority queue is a data structure that assigns each element a priority, which determines the order in which elements will be processed.
        - Usually used for CPU scheduling in operating systems
### 2. Explain the concept of a circular queue? How is it better than a linear queue?
     A circular queue is a data structure that connects the last index to index 0 which solves a memory wastage problem of linear queues. Specifically, how after a deletion, when the front moves forward and the freed space it creates, can't be reused.
### 3. Why do we use multiple queues?
    You need multiple queues when working with different types of data, or process them separately, within the same memory space. The use of multiple queues allows for scheduling and, overall, better organization.

- **Add `A, B, C, D, E, F` to an empty queue** $\rightarrow$ Front - `A | B C | D| E | F` - Rear
- **Delete 2 letters** $\rightarrow$ Front - `C | D | E | F` - Rear
    - FIFO - First In, First Out
- **Add `G`** $\rightarrow$ Front - `C | D | E | F | G` - Rear

- **Add `H`** $\rightarrow$ Front - `C | D | E | F | G | H` - Rear
- **Delete 4 letters** $\rightarrow$ Front - `G | H` - Rear
- **Add `I`** $\rightarrow$ Front - `G | H | I` - Rear
